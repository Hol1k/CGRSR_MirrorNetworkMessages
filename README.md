# Network Messages Subscription Service (Mirror)

## Описание

В проекте реализован сервис обмена сетевыми сообщениями между сервером и клиентами с использованием Mirror.

Основная задача - обойти ограничение Mirror, при котором клиент отключается, если у него не зарегистрирован handler для входящего сообщения. Для этого реализован механизм подписок: сервер отправляет сообщения только тем клиентам, которые явно подписались на их получение.

---

## Основная идея

Используется подход Publisher-Subscriber:

- Клиент отправляет серверу сообщение о подписке на определённый тип сообщений
- Сервер хранит подписки клиентов
- При отправке сообщения сервер проверяет подписки и отправляет его только нужным клиентам

---

## Архитектура

### NetworkSubscriptionService

Центральный сервис, объединяющий серверную и клиентскую логику.

Функции:

- хранение подписок клиентов  
  `Dictionary<NetworkConnectionToClient, HashSet<NetworkMessageType>>`
- обработка сообщений:
  - SubscribeMessage
  - UnsubscribeMessage
  - DisconnectMessage
- отправка сообщений подписчикам
- регистрация handler'ов на клиенте и сервере

---

## Серверная логика

- При запуске сервера регистрируются handler’ы:
  - Subscribe
  - Unsubscribe
  - Disconnect
- Для каждого клиента хранится набор подписок
- При отключении клиента подписки очищаются
- Сообщения отправляются через `conn.Send()` только подписанным клиентам

---

## Клиентская логика

- Регистрируется handler для `HelloMessage`
- Отправляются сообщения:
  - Subscribe
  - Unsubscribe
  - Disconnect
- Полученные сообщения выводятся в UI

---

## Сценарий работы

1. Пользователь запускает Host или Client
2. Клиент подключается к серверу
3. Клиент нажимает кнопку подписки
4. Сервер сохраняет подписку
5. Сервер отправляет сообщение
6. Сообщение получают только подписанные клиенты
7. Клиент отображает сообщение в интерфейсе

---

## UI

### Клиент

- Кнопки подписки и отписки
- Вывод сообщений с отметкой времени

### Сервер

- Поле ввода сообщения
- Кнопка отправки
- Отображение количества получателей

---

## Особенности реализации

- Для хранения подписок используется HashSet:
  - быстрый поиск
  - отсутствие дубликатов
- Не используется `SendToAll`, только точечная отправка через connection
- Обработка отключения клиента:
  - через `NetworkServer.OnDisconnectedEvent`
  - через отдельное сообщение Disconnect
- Регистрация handler’ов выполняется в lifecycle:
  - `OnStartServer`
  - `OnStartClient`

---

## DI

Используется Zenject:

- сервис подписок зарегистрирован как singleton
- UI и сервисы разделены
- зависимости внедряются через контейнер

---

## Запуск

- Start Host - запускает сервер и клиента
- Start Client - подключает клиента к серверу

---

## Результат

Реализована система доставки сообщений, которая:

- предотвращает отключение клиентов
- обеспечивает выборочную отправку сообщений
- разделяет серверную и клиентскую ответственность
